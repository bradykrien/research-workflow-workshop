# Building Your Research Toolkit — Workshop Site

This is a GitHub Pages site for a workshop on digital research tools for humanities grad students.

## How to set up this site on GitHub

### Step 1: Create a new repository
1. Go to [github.com/new](https://github.com/new)
2. Name it something like `research-toolkit-workshop`
3. Set it to **Public** (required for free GitHub Pages)
4. Check **"Add a README file"**
5. Click **Create repository**

### Step 2: Upload these files
1. In your new repo, click **"Add file" → "Upload files"**
2. Drag and drop ALL the files from this folder:
   - `_config.yml`
   - `index.md`
   - `zotero.md`
   - `tropy.md`
   - `literature-monitoring.md`
   - `ai-in-research.md`
   - `mendeley.md`
3. Write a commit message like "Add workshop site files"
4. Click **"Commit changes"**

### Step 3: Enable GitHub Pages
1. Go to your repo's **Settings** tab
2. In the left sidebar, click **"Pages"**
3. Under "Source," select **"Deploy from a branch"**
4. Under "Branch," select **"main"** and leave the folder as **"/ (root)"**
5. Click **Save**
6. Wait 1–2 minutes, then refresh — GitHub will show you your site URL (it'll look like `https://YOUR-USERNAME.github.io/research-toolkit-workshop/`)

### Step 4: Update the config
1. Open `_config.yml` in your repo
2. Click the pencil icon to edit
3. Replace `YOUR-USERNAME` and `YOUR-REPO-NAME` in the footer link with your actual GitHub username and repo name
4. Commit the change

## How to edit the site
- Click any `.md` file in your repo and click the pencil icon to edit
- The site rebuilds automatically after each commit (takes ~60 seconds)
- Pages are written in Markdown — [here's a cheat sheet](https://www.markdownguide.org/cheat-sheet/)
